\doxysubsection{Descripción detallada}
It stores every logged event of a web marketplace as it appears in \href{https://www.kaggle.com/mkechinov/ecommerce-events-history-in-cosmetics-shop}{\texttt{ https\+://www.\+kaggle.\+com/mkechinov/ecommerce-\/events-\/history-\/in-\/cosmetics-\/shop}} It contains the following fields. 


\begin{DoxyItemize}
\item datetime is the time stamp of the record
\item type\+: One f these operations view, cart, remove\+\_\+from\+\_\+cart, purchase
\item prod\+\_\+id\+: Product code. It is an integer value represented as a string, that is, like \char`\"{}0020\char`\"{}. It cannot be null, so if its read is \char`\"{}\char`\"{}, then it must be filled with the E\+M\+T\+P\+Y\+\_\+\+F\+I\+E\+LD constant
\item cat\+\_\+cod\+: The code of the category of the product involved in the event
\item brand\+: The name of the brand
\item price\+: the price of the product expressed as a double value
\item user\+\_\+id\+: A code of the user that performs the operation. I cannot be a null value
\item session\+: Id of the browser session in which the event took place
\end{DoxyItemize}

When these fields appear in a text row, it must appear in this order, separated by charcter \textquotesingle{},\textquotesingle{} and taking into account that some fields might be null, that is, an empty string \char`\"{}\char`\"{} datetime , type , prod\+\_\+id , cat-\/cod , brand , price, user\+\_\+id, session
